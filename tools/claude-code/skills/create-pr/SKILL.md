# create-pr

Open a pull request on the current repo's forge. Works with GitHub, GitLab
(self-hosted included), Gitea, and Forgejo. Scripts handle the deterministic
pieces (forge detection, git plumbing, CLI dispatch); Claude handles the
judgment calls (commit message, PR title, PR body).

All script invocations below assume you run them from the skill directory:

```
cd .claude/skills/create-pr
```

---

## Phase 1 ‚Äî Detect

```bash
python3 -m scripts.forge_detect
```

Returns JSON with `forge` (`github`/`gitlab`/`gitea`/`unknown`), `host`,
`owner`, `repo`, and whether the matching CLI (`gh`/`glab`/`tea`) is
installed.

**If `forge` is `unknown`:** ask the user which forge they use, then re-run
with `CREATE_PR_FORGE=github|gitlab|gitea` exported in the environment. For
self-hosted Gitea/Forgejo on an unusual hostname, you can also add the host
to `CREATE_PR_GITEA_HOSTS` (comma-separated).

**If `cli_installed` is false:** stop and tell the user. Don't try to
install it yourself ‚Äî the required auth (`gh auth login`, `glab auth login`,
`tea login add`) isn't something you can do non-interactively.

## Phase 2 ‚Äî Preflight

```bash
python3 -m scripts.pr_preflight --forge <forge>
```

Returns JSON with:
- `branch`, `base` ‚Äî current feature branch + detected base branch
- `has_commits_ahead`, `commits_ahead`
- `dirty`, `dirty_status` ‚Äî working tree state
- `upstream`, `needs_push`
- `existing_pr_url` ‚Äî if a PR already exists, report it and stop
- `diff_stat`, `commit_log`, `diff` (truncated at 30k chars)

**Bail-out cases:**
- `branch == base` ‚Äî refuse; PRs must come from a feature branch
- `existing_pr_url` is set **and** `dirty` is false **and** `needs_push` is
  false ‚Äî nothing to add; print the URL and stop
- `existing_pr_url` is null **and** `has_commits_ahead` is false **and**
  `dirty` is false ‚Äî nothing to PR; tell the user and stop

**If `dirty`:** draft a conventional commit message from `dirty_status` and
`git diff HEAD`, then commit. Format: `type(scope): subject` on a single
line. Types: `feat`, `fix`, `docs`, `chore`, `refactor`, `test`, `style`,
`perf`, `build`, `ci`. Keep subject ‚â§72 chars, imperative mood.

```bash
cd <repo>
git add -A
git -c commit.gpgsign=false commit -m "<message>"
```

Only disable GPG signing if the user's config requires it and signing is
failing. Prefer to let signing happen normally.

Re-run preflight after committing if you need fresh diff data.

**If `existing_pr_url` is set and there are new commits or dirty changes:**
commit any dirty work (as above), then push the branch and report the
existing URL ‚Äî do NOT draft a new title/body or create a duplicate PR.

```bash
cd <repo>
git push
```

After the push, post a short comment summarising what this push added
(1‚Äì3 sentences; list commit subjects if there are several) so the PR's
activity log reflects the update:

```bash
python3 -m scripts.pr_comment --forge <forge> --body "<summary>"
```

The script returns `{success, supported, error}`. If `supported` is false
(e.g. older `tea`), silently skip ‚Äî it's a courtesy, not a requirement.
If `success` is false but `supported` is true, surface the error.

Then print the `existing_pr_url` and stop. Skip Phases 3‚Äì4 entirely.

## Phase 3 ‚Äî Draft title and body

(Skip this phase if `existing_pr_url` is already set ‚Äî you're updating, not
creating.)

Using the preflight output (`diff_stat`, `commit_log`, `diff`), draft:

**Title** ‚Äî conventional-commit style: `type(scope): subject`. Types:
`feat`, `fix`, `docs`, `chore`, `refactor`, `test`, `style`, `perf`,
`build`, `ci`. Scope is optional but encouraged when the change is
confined to one area (e.g. `feat(auth): ‚Ä¶`). Subject ‚â§72 chars,
imperative mood, lowercase first word, no trailing punctuation. Prefer
summarizing the *intent* ("feat(forge): add GitLab detection") over the
mechanics ("chore: update 3 files"). If there's a single commit ahead
with a well-formed conventional-commit subject, reuse it as the title ‚Äî
tighten it if needed. If the scope is unclear, omit it rather than
inventing one.

**Body** ‚Äî brief markdown. Include:
- A 1‚Äì3 sentence summary of the change and why.
- A bulleted list of the notable things changed, if more than one.
- A "Testing" section only if the change has a testable surface and you
  actually ran the tests. Don't fabricate a checklist.

Keep it tight. No emoji, no "Generated by Claude" footer, no boilerplate
headers for empty sections.

Don't ask for approval ‚Äî draft the title/body and proceed straight to
Phase 4. Report the URL when it's done; the user can edit the PR body on
the forge if they want changes.

## Phase 4 ‚Äî Create

```bash
python3 -m scripts.pr_create \
  --forge <forge> \
  --title "<title>" \
  --body "<body>" \
  [--base <base>] \
  [--draft]
```

The script will:
1. Push the current branch to `origin` (skip with `--skip-push` if already
   pushed).
2. Check if a PR already exists ‚Äî if so, return its URL.
3. Create the PR via the forge's CLI.
4. Print JSON: `{success, number, url, existed, error}`.

Pass `--dry-run` to print what would happen without executing ‚Äî useful when
verifying argument construction.

**On success:** report the URL to the user.
**On failure:** report the error verbatim. Common causes:
- `gh`/`glab`/`tea` not authenticated ‚Üí user runs the login command
- Remote branch diverged ‚Üí user pulls/rebases first
- Base branch doesn't exist on remote ‚Üí check `--base`

## What Claude owns

- Deciding when to commit vs. bail (dirty tree with unrelated WIP ‚Üí ask)
- Writing the commit message, PR title, and PR body
- Interpreting script errors and recovering or escalating

## What scripts own

- Parsing `git remote` URLs and classifying the forge
- Gathering diff/log/status data
- Constructing and invoking the forge CLI
- Detecting and reporting existing PRs

## Environment overrides

- `CREATE_PR_FORGE=github|gitlab|gitea` ‚Äî force forge type regardless of host
- `CREATE_PR_BASE=<branch>` ‚Äî force base branch (skip `origin/HEAD` lookup)
- `CREATE_PR_GITEA_HOSTS=host1,host2` ‚Äî self-hosted Gitea hostnames
- `CREATE_PR_FORGEJO_HOSTS=host1,host2` ‚Äî self-hosted Forgejo hostnames
